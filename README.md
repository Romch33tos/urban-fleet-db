# Информационная система автопредприятия города

## Описание проекта
В данном репозитории представлена реализация реляционной базы данных для управления деятельностью муниципального транспортно-логистического предприятия на базе СУБД Oracle. Информационная система включает в себя комплексное моделирование организационной структуры, учет подвижного состава, управление персоналом, логистику маршрутов, ведение ремонтных ведомостей и мониторинг ключевых эксплуатационных показателей.

## Основные функции системы
Городское автопредприятие осуществляет пассажирские и грузовые перевозки на основе следующих регламентов:
* **Организационная структура** - в базе данных отображена иерархия подразделений, включая цеха, производственные участки и автогаражи (на примере ПАТП и депо города Кемерово).
* **Управление кадрами** - реализован учет сотрудников с разделением на категории: водительский состав, ремонтные и обслуживающие бригады, административное руководство.
* **Автопарк предприятия** - поддерживается учет различных типов транспорта (автобусы, такси, троллейбусы). Реализована связь «многие со многими», позволяющая закреплять один автомобиль за несколькими сменными водителями.
* **Эксплуатация и маршруты** - пассажирский транспорт жестко привязан к городским маршрутам. Система обеспечивает непрерывный учет пассажиропотока и пробега транспортных средств.
* **Ремонтная деятельность** - ведется детальный учет ремонтных работ, включающий информацию о видах неисправностей, временных рамках и финансовых затратах на обслуживание автопарка.

### 1. Архитектура и управление схемой
* **`drop_tables.sql`** - скрипт полного удаления таблиц и сброса схемы с каскадным удалением ограничений (вызывается перед повторным развертыванием).
* **`create_tables.sql`** - описание DDL-структуры всех таблиц, создание первичных и внешних ключей.

### 2. Первоначальное наполнение (DML / Seed)
Данные скрипты используют параметризованные вставки и должны выполняться строго друг за другом:
1. **`insert_division.sql`** - справочник организационных подразделений предприятия.
2. **`insert_employee.sql`** - учетные записи персонала и водительские квалификации.
3. **`insert_vehicle.sql`** - реестр транспортных средств и подвижного состава.
4. **`insert_route.sql`** - городская маршрутная сеть и протяженность рейсов.
5. **`insert_repair.sql`** - журнал технического обслуживания и ремонтов транспорта.
6. **`insert_statistic.sql`** - периодическая отчетность (пассажиропоток, пробег, затраты).
7. **`insert_assignment.sql`** - оперативное распределение ресурсов (водитель - транспорт - маршрут).

### 3. Диагностика и аналитика
* **`describe_tables.sql`** - проверка типов данных и структуры полей созданных таблиц командами `DESC`.
* **`show_constraints.sql`** - аудит целостности схемы и системных ограничений через каталог `user_constraints`.
* **`select_data.sql`** - комплекс скриптов, включающий базовые проверочные выборки, 10 типовых операционных задач и 3 сложных контрольных аналитических запроса.

### 4. Автоматизированный запуск
* **`main_script.sql`** - сквозной управляющий скрипт, объединяющий в себе весь цикл развертывания базы данных (сброс, создание, наполнение и финальное тестирование).

## Технологический стек
* **СУБД** - Oracle DB 11g
* **CASE-средства** - Visual Paradigm (проектирование ERD)
